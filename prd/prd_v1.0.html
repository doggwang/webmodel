0. 产品边界

关键要点

  • • v1.0 是完整闭环产品，包含上传、转换、分享、查看全流程
  • • 最大支持 500MB .skp 文件，自动转换为 glTF/Draco 格式

v1.0 产品范围定义

v1.0 是一个完整的"上传-转换-分享-viewer"闭环产品，而非单纯的查看器工具。

包含范围

  • 模型上传：支持 .skp 格式文件上传，自动转换为 glTF/Draco 压缩格式
  • 分享设置：设置有效期、密码保护、生成分享链接
  • 3D 查看器：Web 端直接渲染，支持旋转、平移、缩放
  • 测量工具：距离测量、面积测量，结果保留并编号
  • 3D 热点：点击显示信息面板，支持面积、材质、图片信息
  • 场景切换：预设场景快速切换，支持客餐厅、主卧等场景
  • 异常处理：加载失败、链接过期、设备性能不足等异常场景

不包含范围

  • 模型编辑：不支持在线编辑 SketchUp 模型
  • 多文件管理：不支持批量上传、文件夹管理
  • 用户系统：不支持用户注册、登录、权限管理
  • 协作功能：不支持多人协作、评论、标注
  • 导出功能：不支持导出为其他格式（如图片、PDF）
  • 后台管理：不支持后台配置场景、热点数据（v1.0 为硬编码）
  • 支付功能：不支持付费、订阅、高级功能

v1.0 核心交付物

一个可独立运行的 Web 应用，设计师上传 .skp 文件后，可生成分享链接，甲方通过链接在浏览器中查看 3D 模型、使用测量工具、查看 3D 热点信息。

一、需求背景

在建筑、室内设计等行业，设计师向甲方汇报方案时，常面临以下痛点：

  • 查看门槛高：甲方电脑通常未安装 SketchUp，手机端官方 Viewer 操作繁琐且加载慢。
  • 沟通效率低：传统的截图或视频无法让甲方自由探索空间，导致沟通中对尺寸、指标的理解偏差。
  • 设备限制：大型模型在移动端容易卡顿甚至崩溃，无法随时随地进行演示。

二、需求目标

关键指标

  • • 首屏加载：≤ 5秒（100MB内模型）
  • • 热点点击准确率：100%
  • • 解析成功率：> 98%
  • • 移动端帧率：≥ 30 FPS

5.1 测量工具交互逻辑

触发流程：

  1. 用户点击顶部工具栏的【测量】图标。
  2. 光标变为十字准心（Crosshair），提示"请选择起点"。
  3. 进入测量模式后锁定镜头操作，点击动作只用于选择模型点位。
  4. 点击模型表面第一个点后提示"请选择终点"，并显示首个锚点。
  5. 点击第二个点，系统通过射线命中模型面，连接两点显示物理距离。
  6. 测量完成后线段与长度标签持续保留，并按 M1、M2... 自动编号。
  7. 点击任一测量线会出现删除按钮，用户可按需删除单条测量数据。
  8. 切换至【面积测量】后，按点选方式绘制多边形，3 个点起算、4 点及以上自动做平面拟合后计算面积。
  9. 面积结果按 A1、A2... 编号持久保留，采用红色边线显示，点击可选中并删除。
  10. 交互优化：在测量模式下可直接点击旋转或平移按钮，系统会自动保存当前测量结果并切换视角模式。
  11. 一键删除：工具栏提供"删除所有测量"按钮，可一键清除所有测量结果。
  12. "点击面计算面积"能力归入后续版本，本版不在界面展示。

5.5 数据来源与配置

v1.0 数据配置方式：

  • 场景数据：从 SketchUp 模型中自动提取预设场景（Scenes），无需手动配置
  • 热点数据：v1.0 采用硬编码方式，在代码中直接定义热点位置和信息
  • 材质信息：从 SketchUp 模型中自动提取材质名称和属性
  • 面积数据：在热点配置中手动输入面积数值
  • 图片信息：在热点配置中手动添加图片URL

数据配置流程：

  1. 设计师在 SketchUp 中创建模型并设置场景（Scenes）
  2. 上传 .skp 文件到系统
  3. 系统自动提取场景数据
  4. 开发人员在代码中硬编码热点数据（位置、面积、材质、图片）
  5. 用户访问分享链接时，系统加载模型和热点数据

v1.0 限制：

  • 不支持后台配置界面，热点数据需要开发人员手动修改代码
  • 不支持从 SketchUp 导出热点数据，需要手动输入
  • 不支持动态修改热点信息，需要重新部署
  • 场景数据完全依赖 SketchUp 中的预设场景

后续版本规划：

  • v1.1：支持从 SketchUp 插件导出热点数据
  • v1.2：提供后台配置界面，支持在线编辑热点信息
  • v1.3：支持批量导入热点数据和场景配置

七、异常与边界处理

异常处理概览

  • • 网络中断：显示错误提示 + 刷新按钮
  • • 设备性能不足：显示性能警告，建议使用电脑端
  • • 链接过期：显示过期提示，无法访问模型
  • • 文件过大：超过500MB立即拒绝
  • • 格式不支持：非.skp文件立即拒绝
  • 网络中断：加载模型过程中断网，显示"连接超时，请检查网络后重试"并提供刷新按钮。
  • 设备性能不足：检测到显存不足时，提示"当前模型复杂度较高，建议使用电脑端查看"。
  • 链接过期：点击已过期的链接，显示"该分享链接已失效"。

移动端适配说明

布局适配

  • 单列布局：移动端采用单列垂直布局，所有内容垂直堆叠
  • 全宽组件：按钮、输入框等组件在移动端占满容器宽度
  • 底部导航：场景按钮在移动端显示在底部，支持横向滚动
  • 工具栏位置：测量工具栏在移动端固定在底部，方便单手操作

👆 交互适配

  • 触摸目标：所有可点击元素最小尺寸为44x44px
  • 手势支持：支持单指旋转、双指平移、捏合缩放
  • 触摸反馈：点击时显示视觉反馈（高亮、涟漪效果）
  • 悬停状态：移动端不依赖悬停状态，所有交互通过点击触发
